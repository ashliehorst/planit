Planit

Event awareness just got rounder!

Planit is an event awareness application designed and developed for mobile-first use. It currently uses data from BYU-Idaho for its events.

Go to Application

What is it?

Planit is an events application developed for BYU-Idaho, and designed for a larger scale application.

What does it do?

Plan is made to help people in an area find events that may interest them. After performing numerous surveys, design iterations, and user tests, we have come up with a prototype that we are happy with. Here’s what you can do with it:

Time Sorting

Select a day.

Days are available for seven days at a time.

Indicate a sorting method

You may choose to sort by time or by the cost of the event. Currently the event price is made up.

See the details

Click or touch on an event to see the details present.

The details available here are:

  • Event description
  • Start time
  • End time
  • Location

Quick Date Selection

We discovered that most people do not plan further out than 2 weeks for activities they are interested in. Hence, we chose to have a quick way to get access to the events for two weeks.

Instead of choosing a date in a drop down form, we chose to create a date slider. This provides fast access for mobile and tablet users to be able to adjust the date they are wanting.

Categorical Labeling

We would love to be able to search by category, but we found that it was too ambiguous. Instead, our events are labeled with categorical icons to express what the category belongs to.

Mobile First

We believe in designing for the mobile version of this application first.

Currently the tablet and desktop designs and developments are not available.

Sorting by Time & Price

We discovered that there was a unanimous agreement among user (based on our testing) that the time and cost of the event were the most important (followed closely by location).

Hence, we chose to develop the application using objects that could work quickly to be sorted on the client side.

Once one of the sorting buttons have been selected, any new date selected will have the sort feature applied to it for the selected button.

Current BYU-Idaho Events

We wanted a way to find activities to do on campus in a fast way. Hence, the events that are in this application are from BYU-Idaho.

The event information is extracted from the RSS feed of the BYU-Idaho calendar, saved as events to our database, and loaded up into our application as objects.

What lies in its future?

We have made a great deal of progress on this application thus far, but it has a way to go before it’s ready for the big leagues. Here’s what we are interested in developing for it:

Events from Facebook

The point of this application is to enhance event awareness. To do this, it will be wise for us to import public events into our system. That way you can start advertising that garage sale of yours somewhere besides CraigsList!

Save Events to Calendar

We would love for our system to be able to not only allow our users to view events, but save them to their calendar! Get all of the monotonous data entry stuff out of the way!
